# AI-SKILL 内部核心团队

> 本文档明确项目内部专家团队的职责分工。所有计划任务由核心团队直接完成；社区贡献者可以参与，但不影响项目按期交付。

## 团队定位

- **内部主导**：核心任务、里程碑、质量门禁由本团队负责执行。
- **社区开放**：外部贡献者可以提 issue / PR / 建议，但非阻塞项目进度。
- **一人多岗**：早期阶段，同一专家可能兼任多个角色。

## 决策与执行规则

1. **任务不等待外部人员**：所有计划任务优先由内部专家完成；社区志愿者仅作为补充。
2. **内部 review**：关键改动由对应 area 专家自查或交叉 review，Project Lead 终审。
3. **文档即契约**：职责、流程、验收标准全部写入仓库，避免口头约定。
4. **持续迭代**：每月月初 review 团队分工，根据项目阶段调整角色边界。

## 与社区的关系

- 社区是**参与入口**，不是**交付依赖**。
- 外部贡献者可以通过 [`CONTRIBUTING.md`](../CONTRIBUTING.md) 提交 skill、修复问题、提出建议。
- Maintainer 任命仍保留公开标准，但首批 maintainer 由 Project Lead 直接从内部核心团队任命，无需等待外部贡献者满足 3 PR 条件。
